> 面向评审团的产品技术说明 | v1.0 | 2026-03-15

---

## 一、产品定义

**一句话介绍**：流光·办公副驾是你的飞书数据管家 + 智能问答助手——它自动收集整理你在飞书上的各种数据，用AI帮你快速找到任何需要的信息。

**核心价值**：

- 让散落在飞书各处的信息（文档、会议、聊天）自动汇聚到一个平台
- 用自然语言提问就能获取答案，不再需要逐个翻找
- 自动发现人物、项目之间的关联关系
- 一键生成报告，告别手动汇总

**产品定位**：企业内部数字员工，对齐公司"人机协同"AI战略。

## 二、数据从哪来？——四条数据入口

流光有四种方式获取数据，覆盖日常办公中的主要信息来源：

```
┌─────────────────────────────────────────────────┐
│                  四条数据入口                      │
├─────────────────────────────────────────────────┤
│                                                  │
│  1. 定时自动同步（每30分钟）                       │
│     首次配置飞书多维表格/云文件夹的数据源地址后     │
│     系统自动定时增量拉取最新数据，后续无需人工操作   │
│                                                  │
│  2. 飞书工作配方 + 多维表格（会议/聊天数据）        │
│     员工在飞书配置工作配方，自动将会议纪要、         │
│     聊天摘要写入多维表格，流光从该表格自动同步       │
│                                                  │
│  3. 飞书机器人归档（实时）                         │
│     直接给流光机器人发消息或转发内容，即可归档      │
│     机器人智能识别内容类型，自动分类入库            │
│                                                  │
│  4. 手动上传                                      │
│     支持直接上传 PDF、Word、Excel、CSV 等文件       │
│     上传后自动解析内容、生成索引                    │
│                                                  │
└─────────────────────────────────────────────────┘
```

**实际效果**：首次配置好数据源后，数据自动归集，员工不需要再手动整理和转发信息——所有内容都在一个平台上，搜得到、问得到。

## 三、数据怎么处理？——AI六步流水线

原始数据进入平台后，会经过一条智能"流水线"加工，变成可搜索、可分析的"数据资产"：

```
  原始数据（飞书表格、文档、聊天消息）
          │
          ▼
  ┌──────────────┐
  │ 第1步：清洗    │  去除空白、统一格式
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 第2步：提取    │  识别标题、内容、时间、人员等关键信息
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 第3步：转换    │  AI自动识别数据类型和字段含义（智能映射）
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 第4步：增强    │  AI自动打标签、提取关键词、生成摘要
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 第5步：向量化  │  把文本转为"语义向量"（实现智能搜索的关键）
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 第6步：入库    │  存入数据库，建立搜索索引，构建知识图谱
  └──────────────┘
```

**通俗解释**：就像一个图书管理员，收到一堆杂乱的资料后，先整理干净（清洗），再分类贴标签（提取+增强），然后编好目录方便查找（向量化+入库）。

**智能映射的价值**：不同部门的飞书表格字段名千差万别（有人写"负责人"，有人写"项目经理"），AI能自动识别它们其实是同一个意思，统一归类——这避免了大量人工整理工作。

## 四、四大AI能力

平台集成了四大AI能力，各有分工：

### 能力一：智能字段映射

**做什么**：AI自动识别飞书表格中每一列的含义，将其对应到系统标准字段。

**为什么重要**：不同团队建的飞书表格字段名各不相同，人工逐一映射既耗时又容易出错。AI能在几秒内完成这项工作，且结果可缓存复用。

### 能力二：语义搜索

**做什么**：不需要记住精确的关键词，用自然语言描述就能找到相关内容。比如搜"最近讨论过的市场策略"，即使文档中没有"市场策略"四个字，但内容是关于市场策略的，也能被找到。

**为什么重要**：传统搜索只能匹配"字面一样"的内容，而语义搜索能理解"意思相近"的内容，搜索成功率大幅提升。

### 能力三：知识图谱

**做什么**：AI从文档和会议中自动提取人物、项目、组织等实体，以及它们之间的关系（谁负责什么项目、谁参加了哪个会议），构建一张可视化的"知识网络"。

**为什么重要**：让隐藏在文字中的人际关系和项目关联变得一目了然，帮助管理层快速了解团队协作全貌。

### 能力四：智能问答

**做什么**：像跟同事聊天一样提问，AI从你有权限看到的数据中找到答案，并标注信息来源。

**为什么重要**：这是产品的核心价值——把"翻找15分钟"变成"提问5秒得到答案"。每个回答都有据可查，点击来源即可跳转原始内容。

```
┌──────────────────────────────────────────────────┐
│                   AI能力全景                       │
├──────────┬──────────────────┬─────────────────────┤
│  能力类型  │     使用的模型     │       做什么          │
├──────────┼──────────────────┼─────────────────────┤
│ 文本理解   │ GLM-4.6-air      │ 字段映射、信息提取    │
│ 与生成     │                  │ 标签推荐、摘要生成    │
├──────────┼──────────────────┼─────────────────────┤
│ 智能对话   │ GLM-5            │ AI问答、报告生成      │
│ (Agent)   │                  │ 日历助理              │
├──────────┼──────────────────┼─────────────────────┤
│ 语义向量   │ BAAI/bge-m3      │ 文本转向量            │

├──────────┼──────────────────┼─────────────────────┤
│ 视觉识别   │ Qwen3-VL-8B     │ 识别图片中的文字      │
├──────────┼──────────────────┼─────────────────────┤
```

## 五、智能问答是怎么工作的？

这是产品最核心的能力，采用业界主流的 **RAG（检索增强生成）** 技术：

```
  用户提问："上周XX项目有什么进展？"
          │
          ▼
  ┌──────────────────────────┐
  │ 第1步：理解问题            │
  │ AI分析用户意图和关键词     │
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第2步：双通道检索           │
  │ ① 向量检索：找语义相似的    │ ← "意思接近"的内容
  │ ② 关键词检索：找精确匹配的  │ ← "字面一样"的内容
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第3步：融合排序             │
  │ 把两个通道的结果合并排序    │ ← 取最相关的Top结果
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第4步：权限过滤             │
  │ 只返回用户有权限看到的数据  │ ← 数据安全保障
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第5步：AI生成回答           │
  │ 基于检索到的资料，生成回答  │ ← 附带来源引用
  │ 通过流式传输实时输出        │ ← 逐字显示，体验流畅
  └──────────────────────────┘
```

**关键优势**：

- **有据可查**：每个回答都标注了信息来源，点击即可跳转到原始文档
- **权限安全**：不同人问同一个问题，可能因权限不同而得到不同答案——AI绝不会泄露用户无权看到的数据
- **双通道检索**：语义理解 + 关键词精确匹配，两种方式互补，确保搜索质量

## 六、数据安全设计

数据安全是企业级产品的生命线。流光设计了三道安全防线：

```
┌──────────────────────────────────────────────────┐
│                  数据安全三道防线                   │
├──────────────────────────────────────────────────┤
│                                                   │
│  第一道：身份认证                                   │
│  └→ 飞书SSO单点登录，只有企业飞书账号才能访问       │
│  └→ 非企业成员完全无法进入系统                      │
│                                                   │
│  第二道：行级权限控制                               │
│  └→ 每条数据都有"归属人"标记                       │
│  └→ 默认只能看到自己的数据                          │
│  └→ 可按部门或指定人员灵活分享                      │
│  └→ 管理员可查看全部数据                            │
│                                                   │
│  第三道：AI回答权限感知                             │
│  └→ AI在检索资料时，自动过滤无权限的数据            │
│  └→ 不同人问同一个问题，可能得到不同的回答           │
│  └→ 确保AI不会成为数据泄露的通道                    │
│                                                   │
└──────────────────────────────────────────────────┘
```

## 八、系统架构概览

采用经典三层架构，通过Docker容器化部署：

```
                    用户浏览器
                        │
                        ▼
              ┌──────────────────┐
              │   前端服务(Nginx)  │  ← 端口 80
              │   React 网页应用   │     17个功能页面
              └────────┬─────────┘
                       │ /api 请求转发
                       ▼
              ┌──────────────────┐
              │  后端服务(FastAPI) │  ← 端口 8000
              │  26个API模块      │     异步Python
              │  8项定时任务      │     后台自动运行
              └────────┬─────────┘
                       │ 数据读写
                       ▼
              ┌──────────────────┐
              │  数据库(PostgreSQL)│  ← 端口 5432
              │  + 向量检索扩展    │     19张数据表
              └──────────────────┘

外部依赖：
  飞书API（用户认证、数据读取、任务同步、机器人归档）
  AI大模型API（智能问答、字段映射、信息提取）
```

**部署方式**：`docker-compose up -d` 一键启动，包含数据库、后端、前端三个服务，启动时自动完成数据库结构迁移。

## 九、定时任务体系

系统包含8项自动运行的后台任务，实现"无人值守"的数据处理：

| 任务 | 运行频率 | 做什么 |
|------|---------|--------|
| ETL数据同步 | 每30分钟 | 从飞书多维表格增量拉取最新数据 |
| 云文件夹同步 | 每30分钟 | 扫描飞书云文件夹中的新文件 |
| 待办自动提取 | 每60分钟 | AI从新数据中识别行动项，生成待办 |
| 待办状态同步 | 每30分钟 | 与飞书任务双向同步待办状态 |
| 知识图谱构建 | 每120分钟 | AI从数据中提取人物、项目等实体和关系 |
| 结构化表同步 | 每120分钟 | 同步飞书多维表格的行数据 |
| 人物画像生成 | 每180分钟 | 分析用户沟通行为，生成画像报告 |
| 日程提醒检查 | 每5分钟 | 检查即将开始的日程，推送提醒 |

所有任务均支持通过管理后台手动触发。

## 十一、风险与合规

| 风险类型 | 风险描述 | 应对措施 |
|---------|---------|---------|
| 数据隐私 | 飞书数据包含员工敏感信息 | 行级权限控制，数据不出企业环境，日志脱敏 |
| AI幻觉 | LLM可能生成不准确的内容 | RAG机制确保回答基于真实数据，附带来源引用供验证 |
| 模型依赖 | 依赖外部AI模型API | OpenAI兼容接口可随时切换模型，不绑定单一供应商 |
| 数据质量 | 飞书原始数据质量参差不齐 | ETL流水线包含清洗和校验步骤，支持自定义清洗规则 |
| 系统可用性 | 服务异常影响用户使用 | Docker容器化部署+健康检查+飞书Webhook告警 |
