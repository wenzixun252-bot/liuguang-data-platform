## 一、为什么需要流光？
### 1.1 现状痛点
员工日常工作中，信息散落在飞书的文档、会议纪要、群聊、多维表格等多个角落：
```
  现在的工作方式（没有流光）：

  "上周项目进展怎么样？"
       │
       ├── 翻飞书文档…… 没找到
       ├── 搜群聊记录…… 关键词想不起来
       ├── 问同事…… 同事也记不清
       ├── 找会议纪要…… 不知道在哪个群
       └── 最终拼凑出答案 ——> 耗时15-30分钟

  有了流光之后：

  "上周项目进展怎么样？"
       │
       └── 在流光AI助手中提问
            → 3秒内从文档+会议+聊天中综合检索
            → 给出答案 + 标注每条信息来源
            → 点击来源可直接跳转原文 ——> 耗时1分钟
```
### 1.2 量化价值（ROI估算）
基于30人核心团队试点的保守估算：
| 场景 | 现状 | 使用流光后 | 年化节省 |
|------|------|-----------|---------|
| 信息检索 | 每人每天1.5小时翻找 | 0.5小时 | **7,500工时** |
| 报告编制 | 管理层每份2小时 | AI生成5分钟 | **1,000工时** |
| 会议待办跟进 | 手动记录易遗漏 | AI自动提取 | **750工时** |
| **合计** | — | — | **≈9,250工时/年（≈92.5万元）** |
### 1.3 为什么不用飞书自带的搜索？
| 维度 | 飞书原生搜索 | 流光·办公副驾 |
|------|------------|-------------|
| 搜索范围 | 仅飞书内单一模块 | 文档+会议+聊天+表格，跨源聚合 |
| 搜索方式 | 关键词精确匹配 | 语义理解+关键词，说大白话也能搜到 |
| 知识图谱 | 无 | 自动发现人物-项目-组织关系 |
| 报告生成 | 无 | AI一键生成周报/月报 |
| 待办提取 | 手动记录 | AI从会议纪要自动提取 |
| 数据权限 | 平台级粗粒度 | 行级权限，AI回答也经过滤 |
| 数据存储 | 数据在飞书 | 私有化部署，数据不出企业 |
**核心差异**：飞书搜索是"在哪找"的问题，流光解决的是"自动收集+AI理解+跨源回答"的问题。
## 二、产品定义
**一句话介绍**：流光·办公副驾是你的飞书数据管家 + 智能问答助手——它自动收集整理你在飞书上的各种数据，用AI帮你快速找到任何需要的信息。
**核心价值**：
- 让散落在飞书各处的信息（文档、会议、聊天）自动汇聚到一个平台
- 用自然语言提问就能获取答案，不再需要逐个翻找
- 自动发现人物、项目之间的关联关系
- 一键生成报告，告别手动汇总
**产品定位**：企业内部数字员工，对齐公司"人机协同"AI战略。
## 三、数据从哪来？——四条数据入口
流光有四种方式获取数据，覆盖日常办公中的主要信息来源：
```
┌─────────────────────────────────────────────────┐
│                  四条数据入口                      │
├─────────────────────────────────────────────────┤
│                                                  │
│  1. 定时自动同步（每30分钟）                       │
│     首次配置飞书多维表格/云文件夹的数据源地址后     │

│                                                  │
│  2. 飞书工作配方 + 多维表格（会议/聊天数据）        │
│     员工在飞书配置工作配方，自动将会议纪要、         │
│     聊天摘要写入多维表格，流光从该表格自动同步       │
│                                                  │
│  3. 飞书机器人归档（实时）                         │
│     直接给流光机器人发消息或转发内容，即可归档      │

│                                                  │
│  4. 手动上传                                      │

```
**实际效果**：首次配置好数据源后，数据自动归集，员工不需要再手动整理和转发信息——所有内容都在一个平台上，搜得到、问得到。
## 四、数据怎么处理？——AI六步流水线
原始数据进入平台后，会经过一条智能"流水线"加工，变成可搜索、可分析的"数据资产"：
```
  原始数据（飞书表格、文档、聊天消息）
          │
          ▼
  ┌──────────────┐
  │ 第1步：清洗    │  去除空白、统一格式（可配置清洗规则）
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 第2步：提取    │  识别标题、内容、时间、人员等关键信息
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 第3步：转换    │  AI自动识别数据类型和字段含义（智能映射）
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 第4步：增强    │  AI自动打标签、提取关键词、生成摘要
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 第5步：向量化  │  把文本转为"语义向量"（实现智能搜索的关键）
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 第6步：入库    │  存入数据库，按提取规则提取关键信息，构建知识图谱
  └──────────────┘
```
**通俗解释**：就像一个图书管理员，收到一堆杂乱的资料后，先整理干净（清洗），再分类贴标签（提取+增强），然后编好目录方便查找（向量化+入库）。
**智能映射的价值**：不同部门的飞书表格字段名千差万别（有人写"负责人"，有人写"项目经理"），AI能自动识别它们其实是同一个意思，统一归类——这避免了大量人工整理工作。
### 4.1 数据加工配方——提取规则与清洗规则
流光的一大核心亮点是**用户可自定义数据加工"配方"**，而不是一刀切地处理所有数据。系统提供两种可配置的规则引擎：
#### 提取规则——告诉AI"从文档里挖什么"
```
  举个例子：能源部门导入了一批项目文档

  第1步：选择行业板块——"能源"
         系统自动推荐字段：装机容量、投资金额、项目阶段、合作方…

  第2步：根据需要增删字段
         + 自定义字段："风险等级"（描述：识别文中提到的风险等级）

  第3步：保存并激活规则

  之后，每次导入新文档/会议纪要时：
  ┌──────────────────────────────────────────┐
  │  AI自动阅读每篇文档，按规则提取关键信息：    │
  │                                            │
  │  文档A《XX光伏项目可研报告》                 │
  │  → 装机容量：100MW                          │
  │  → 投资金额：5亿元                          │
  │  → 项目阶段：可研阶段                       │
  │  → 风险等级：中等                            │
  │                                            │
  │  文档B《XX风电项目会议纪要》                  │
  │  → 装机容量：50MW                           │
  │  → 投资金额：2亿元                          │
  │  → 合作方：XX新能源公司                     │
  │  → 风险等级：低                             │
  └──────────────────────────────────────────┘

  提取结果可汇总查看、搜索、导出Excel
```
**4种行业板块模板**：平台内置能源、城乡建设、不良资产、其他4种板块，每种预设了该行业常用的提取字段。用户也可以完全自定义字段——只需填写字段名称和描述，AI就知道该提取什么。
#### 清洗规则——告诉系统"怎么整理表格数据"
```
  举个例子：财务部门导入了一张多维表格

  表格原始状态（常见问题）：
  ┌──────┬────────────┬──────────┬─────┬─────┬─────┐
  │ 项目  │ 金额        │ 日期      │ 省   │ 市   │ 区   │
  ├──────┼────────────┼──────────┼─────┼─────┼─────┤
  │ 项目A │ 1,234.56万元│ 2025/3/1 │ 四川 │ 成都 │ 高新 │
  │ 项目B │ 500万      │ 2025年3月 │ 四川 │ 成都 │ 武侯 │
  │       │            │          │     │     │     │  ← 空行
  │ 项目A │ 1,234.56万元│ 2025/3/1 │ 四川 │ 成都 │ 高新 │  ← 重复行
  └──────┴────────────┴──────────┴─────┴─────┴─────┘

  启用清洗规则后（一键自动处理）：
  ┌──────┬──────────┬────────────┬────────────┐
  │ 项目  │ 金额      │ 日期        │ 地区        │
  ├──────┼──────────┼────────────┼────────────┤
  │ 项目A │ 12345600 │ 2025-03-01 │ 四川成都高新 │
  │ 项目B │ 5000000  │ 2025-03-01 │ 四川成都武侯 │
  └──────┴──────────┴────────────┴────────────┘

  变化：
  ✓ 金额格式统一（去除逗号和"万元"单位，转为标准数值）
  ✓ 日期格式统一（YYYY-MM-DD）
  ✓ 空行自动删除
  ✓ 重复行自动去除
  ✓ "省""市""区"三列智能合并为"地区"（LLM自动完成）
```
**8种清洗选项一览**：
| 选项 | 类型 | 做什么 |
|------|------|--------|
| 去重 | 规则化 | 基于内容指纹去除完全重复的行 |
| 空行处理 | 规则化 | 删除空值比例超过阈值的行（阈值可调0%-100%） |
| 日期标准化 | 规则化 | 识别多种日期格式，统一转换为YYYY-MM-DD |
| 数值标准化 | 规则化 | 去除千分位逗号、单位文字（万元/亿/%），保留纯数值 |
| 空白处理 | 规则化 | 去除多余空格、换行符等空白字符 |
| AI智能合并 | LLM驱动 | 自动识别并合并含义相同的列（如"联系电话"和"手机号"） |
| AI智能清洗 | LLM驱动 | 修正错别字、统一表述风格（如"完成了""done"→"已完成"） |
| 自定义指令 | LLM驱动 | 用户描述想要的操作（如"合并省市区为地区"），AI自动执行 |
**设计亮点**：清洗分为两层——前5项是**确定性规则**（快速、100%准确），后3项是**AI智能清洗**（灵活、处理复杂场景）。两层配合确保既快速又智能。
## 五、四大AI能力
平台集成了四大AI能力，各有分工：
### 能力一：智能字段映射
**做什么**：AI自动识别飞书表格中每一列的含义，将其对应到系统标准字段。
**为什么重要**：不同团队建的飞书表格字段名各不相同，人工逐一映射既耗时又容易出错。AI能在几秒内完成这项工作，且结果可缓存复用。
### 能力二：语义搜索
**做什么**：不需要记住精确的关键词，用自然语言描述就能找到相关内容。比如搜"最近讨论过的市场策略"，即使文档中没有"市场策略"四个字，但内容是关于市场策略的，也能被找到。
**为什么重要**：传统搜索只能匹配"字面一样"的内容，而语义搜索能理解"意思相近"的内容，搜索成功率大幅提升。
### 能力三：知识图谱
**做什么**：AI从文档和会议中自动提取人物、项目、组织等实体，以及它们之间的关系（谁负责什么项目、谁参加了哪个会议），构建一张可视化的"知识网络"。
**为什么重要**：让隐藏在文字中的人际关系和项目关联变得一目了然，帮助管理层快速了解团队协作全貌。
### 能力四：智能问答
**做什么**：像跟同事聊天一样提问，AI从你有权限看到的数据中找到答案，并标注信息来源。
**为什么重要**：这是产品的核心价值——把"翻找15分钟"变成"提问5秒得到答案"。每个回答都有据可查，点击来源即可跳转原始内容。
```
┌──────────────────────────────────────────────────┐
│                   AI能力全景                       │
├──────────┬──────────────────┬─────────────────────┤
│  能力类型  │     使用的模型     │       做什么          │
├──────────┼──────────────────┼─────────────────────┤
│ 文本理解   │ GLM-4.6-air      │ 字段映射、信息提取    │
│ 与生成     │                  │ 标签推荐、摘要生成    │
├──────────┼──────────────────┼─────────────────────┤
│ 智能对话   │ GLM-5            │ AI问答、报告生成      │
│ (Agent)   │                  │ 日历助理              │
├──────────┼──────────────────┼─────────────────────┤
│ 语义向量   │ BAAI/bge-m3      │ 文本转向量            │
│(Embedding)│                  │ 实现语义搜索          │
├──────────┼──────────────────┼─────────────────────┤
│ 视觉识别   │ Qwen3-VL-8B     │ 识别图片中的文字      │
├──────────┼──────────────────┼─────────────────────┤
│ 语音转文字  │ FunASR          │ 音频转写为文本        │
```

## 六、智能问答是怎么工作的？
这是产品最核心的能力，采用业界主流的 **RAG（检索增强生成）** 技术：
```
  用户提问："上周XX项目有什么进展？"
          │
          ▼
  ┌──────────────────────────┐
  │ 第1步：理解问题            │
  │ AI分析用户意图和关键词     │
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第2步：双通道检索           │
  │ ① 向量检索：找语义相似的    │ ← "意思接近"的内容
  │ ② 关键词检索：找精确匹配的  │ ← "字面一样"的内容
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第3步：融合排序             │
  │ 把两个通道的结果合并排序    │ ← 取最相关的Top结果
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第4步：权限过滤             │
  │ 只返回用户有权限看到的数据  │ ← 数据安全保障
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第5步：AI生成回答           │
  │ 基于检索到的资料，生成回答  │ ← 附带来源引用
  │ 通过流式传输实时输出        │ ← 逐字显示，体验流畅
  └──────────────────────────┘
```
**关键优势**：
- **有据可查**：每个回答都标注了信息来源，点击即可跳转到原始文档
- **权限安全**：不同人问同一个问题，可能因权限不同而得到不同答案——AI绝不会泄露用户无权看到的数据
- **双通道检索**：语义理解 + 关键词精确匹配，两种方式互补，确保搜索质量
## 七、数据安全设计
数据安全是企业级产品的生命线。流光设计了三道安全防线：
```
┌──────────────────────────────────────────────────┐
│                  数据安全三道防线                   │
├──────────────────────────────────────────────────┤
│                                                   │
│  第一道：身份认证                                   │
│  └→ 飞书SSO单点登录，只有企业飞书账号才能访问       │
│  └→ 非企业成员完全无法进入系统                      │
│                                                   │
│  第二道：行级权限控制                               │
│  └→ 每条数据都有"归属人"标记                       │
│  └→ 默认只能看到自己的数据                          │
│  └→ 可按部门或指定人员灵活分享                      │
│  └→ 管理员可查看全部数据                            │
│                                                   │
│  第三道：AI回答权限感知                             │
│  └→ AI在检索资料时，自动过滤无权限的数据            │
│  └→ 不同人问同一个问题，可能得到不同的回答           │
│  └→ 确保AI不会成为数据泄露的通道                    │
│                                                   │
└──────────────────────────────────────────────────┘
```

| LLM调用费用 | Schema映射结果MD5缓存，相同结构不重复调用；按需调用而非全量处理 |
| 服务器成本 | Docker一键部署，单台服务器即可运行全套系统（首批50人规模） |
| 向量存储 | pgvector内嵌于PostgreSQL，无需额外向量数据库服务费用 |
| 维护成本 | 全自动ETL同步+定时任务，日常运行无需人工干预 |
| 模型迁移 | 采用OpenAI兼容接口，可随时切换到性价比更优的模型 |

## 十、定时任务体系
系统包含8项自动运行的后台任务，实现"无人值守"的数据处理：
| 任务 | 运行频率 | 做什么 |
|------|---------|--------|
| ETL数据同步 | 每30分钟 | 从飞书多维表格增量拉取最新数据 |
| 云文件夹同步 | 每30分钟 | 扫描飞书云文件夹中的新文件 |
| 待办自动提取 | 每60分钟 | AI从新数据中识别行动项，生成待办 |
| 待办状态同步 | 每30分钟 | 与飞书任务双向同步待办状态 |
| 知识图谱构建 | 每120分钟 | AI从数据中提取人物、项目等实体和关系 |
| 结构化表同步 | 每120分钟 | 同步飞书多维表格的行数据 |
| 人物画像生成 | 每180分钟 | 分析用户沟通行为，生成画像报告 |
| 日程提醒检查 | 每5分钟 | 检查即将开始的日程，推送提醒 |
所有任务均支持通过管理后台手动触发。
