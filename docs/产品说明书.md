# 流光智能数据资产平台 — 产品说明书

> 版本：v1.0 | 更新日期：2026-03-04

---

## 一、项目简介

**流光（Liuguang）** 是一个企业级智能数据资产平台。简单来说，它能把你在飞书上散落的各种数据（文档、会议纪要、聊天记录等）自动收集、整理、存储起来，然后通过一个智能助手让你用自然语言提问，快速找到需要的信息。

### 核心价值

- **数据自动收集**：定时从飞书多维表格、云文件夹中同步数据，无需手动搬运
- **智能问答**：像跟人聊天一样，问问题就能从海量数据中找到答案
- **权限安全**：每个人只能看到自己有权限的数据，管理员可以看全部
- **知识图谱**：自动发现人物、项目、话题之间的关联关系

## 二、核心功能

### 2.1 飞书登录（SSO）

用户通过飞书扫码或授权登录，平台自动创建账号并分配权限。登录后获得一个 24 小时有效的令牌（JWT），后续所有操作都用这个令牌验证身份。

- 支持两种角色：**普通员工（employee）** 和 **管理员（admin）**
- 系统内置一个超级管理员，不可被降级

### 2.2 ETL 数据同步管线

ETL 是 Extract（抽取）→ Transform（转换）→ Load（加载）的缩写，整个流程自动运行：

```
飞书多维表格 → 抽取新数据 → AI智能转换字段 → 生成向量 → 存入数据库
```

**工作原理：**

1. **抽取（Extract）**：检查上次同步时间，只拉取新增或修改的记录（增量同步）
2. **转换（Transform）**：用 AI 大模型自动识别飞书表格的字段含义，映射到系统标准字段。无法映射的字段会保存到一个灵活的扩展字段中，不会丢失任何数据
3. **加载（Load）**：为每条记录生成文本向量（用于语义搜索），然后存入数据库。如果记录已存在则自动更新

**支持的数据类型：**

| 类型 | 说明 | 典型字段 |
|------|------|----------|
| 文档（Document） | 企业知识文档、制度文件等 | 标题、内容、作者、分类、标签 |
| 会议纪要（Meeting） | 会议记录和决策 | 标题、时间、参与人、议程、结论、行动项 |
| 聊天消息（ChatMessage） | 重要群聊/私聊记录 | 发送人、内容、时间、提及的人 |

### 2.3 流光智能助手（RAG 问答）

RAG 是 Retrieval-Augmented Generation（检索增强生成）的缩写。用户提问后，系统先从数据库中找到相关内容，再让 AI 基于这些内容生成回答。

**搜索方式：**

- **向量搜索**：理解问题的语义，找到含义相近的内容（即使用词不同也能找到）
- **关键词搜索（BM25）**：精确匹配关键词
- **融合排序（RRF）**：把两种搜索的结果合并，取最相关的内容

**权限隔离**：搜索时会自动过滤，确保用户只能搜到自己有权限看的数据。

**支持功能：**
- 流式输出（SSE）：答案像打字一样逐字显示，体验更流畅
- 附件解析：上传 PDF、Word、TXT、CSV 等文件后可基于内容提问
- 对话记忆：支持多轮对话，AI 记得上下文
- 导出到飞书：可将对话导出为飞书文档

### 2.4 知识图谱

系统自动从文档、会议、聊天中提取实体（人物、项目、话题、组织等）和它们之间的关系，构建知识图谱。

**实体类型：** 人物、项目、话题、组织、事件、文档、社群

**关系类型：** 合作关系、参与项目、讨论话题、隶属关系、相关关系

### 2.5 自动待办提取

系统定期扫描新的会议纪要和聊天记录，用 AI 自动识别其中的行动项（比如"张三下周完成方案"），生成待办事项，还可以同步到飞书任务。

**待办状态流转：** 待审核 → 进行中 → 已完成 / 已忽略

### 2.6 报告自动生成

基于指定时间范围和数据源，AI 自动汇总数据并生成报告，支持发布到飞书文档。

### 2.7 人物洞察分析

分析目标用户的沟通风格、关注领域等，生成领导力画像报告。

### 2.8 部门管理

从飞书同步企业部门结构，支持按部门分享数据可见权限。

### 2.9 结构化数据表

支持导入飞书多维表格或本地上传的表格（CSV/Excel），在平台内预览和管理。

### 2.10 日程提醒

同步飞书日历，定时检查即将开始的日程并发送提醒。

## 三、技术架构

### 3.1 整体架构图

```
┌─────────────────────────────────────────────────────┐
│                     前端（React + Vite）              │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP / SSE
                       ▼
┌─────────────────────────────────────────────────────┐
│              后端（FastAPI，异步 Python）              │
│                                                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐  │
│  │ API 路由  │  │ 业务服务  │  │ 定时任务调度器   │  │
│  │ (api/)   │  │(services/)│  │(worker/scheduler)│  │
│  └────┬─────┘  └────┬─────┘  └───────┬──────────┘  │
│       │              │                │              │
│       ▼              ▼                ▼              │
│  ┌──────────────────────────────────────────────┐   │
│  │             数据库层 (SQLAlchemy Async)        │   │
│  └──────────────────────┬───────────────────────┘   │
└─────────────────────────┼───────────────────────────┘
                          │
                          ▼
              ┌───────────────────────┐
              │  PostgreSQL 16        │
              │  + pgvector 扩展      │
              │  (关系数据 + 向量检索) │
              └───────────────────────┘

外部依赖：
  飞书 API（用户认证、数据读取、任务同步）
  DeepSeek / Qwen API（大模型推理、向量生成）
```

### 3.2 技术选型

| 组件 | 技术 | 为什么选它 |
|------|------|-----------|
| Web 框架 | FastAPI | 原生异步支持，自动生成 API 文档，性能优异 |
| 数据库 | PostgreSQL 16 + pgvector | 成熟稳定，pgvector 扩展支持向量搜索，一个库搞定关系数据和向量数据 |
| ORM | SQLAlchemy 2.0 (async) | Python 最主流的 ORM，异步版本性能好 |
| 认证 | 飞书 OAuth + JWT | 企业内用飞书登录最方便，JWT 无状态好扩展 |
| 大模型 | OpenAI 兼容 SDK | 可无缝切换 DeepSeek、Qwen 等国产模型 |
| 搜索 | 向量 + BM25 + RRF | 语义和关键词互补，效果比单一方式好很多 |
| 任务调度 | APScheduler | 轻量级，直接嵌入应用进程，无需额外部署 |
| HTTP 客户端 | httpx | 异步，API 兼容 requests |
| 容器 | Docker + Docker Compose | 一键启动，环境一致 |
| 前端 | React + TypeScript + Vite | 现代前端技术栈，开发体验好 |

### 3.3 代码分层

```
backend/app/
│
├── api/           → 接口层：接收请求，返回响应（类似"前台接待"）
│   └── deps.py    → 公共依赖：数据库连接、用户身份验证
│
├── models/        → 数据模型层：定义数据库表结构（类似"数据库设计图"）
│
├── schemas/       → 数据校验层：定义请求和响应的格式（类似"表单验证规则"）
│
├── services/      → 业务逻辑层：核心业务代码（类似"后厨"）
│   ├── feishu.py  → 飞书 API 调用
│   ├── llm.py     → 大模型调用
│   ├── rag.py     → 智能检索
│   └── etl/       → 数据同步管线
│
├── worker/        → 调度层：定时执行后台任务（类似"定时闹钟"）
│
├── utils/         → 工具层：通用工具函数
│
├── config.py      → 配置：从 .env 文件读取所有配置项
├── database.py    → 数据库连接管理
└── main.py        → 应用入口
```

## 六、定时任务

系统通过 APScheduler 自动执行以下后台任务：

| 任务名 | 执行间隔 | 说明 |
|--------|---------|------|
| ETL 数据同步 | 30 分钟 | 从飞书多维表格增量同步数据 |
| 云文件夹同步 | 30 分钟 | 同步飞书云文件夹中的文件 |
| 待办自动提取 | 60 分钟 | 从新数据中识别待办事项 |
| 待办状态同步 | 30 分钟 | 与飞书任务双向同步状态 |
| 知识图谱构建 | 120 分钟 | 从新数据中提取实体和关系 |
| 结构化表同步 | 120 分钟 | 同步飞书多维表格数据 |
| 人物画像生成 | 180 分钟 | 分析用户行为生成画像 |
| 日程提醒检查 | 5 分钟 | 检查即将开始的日程并提醒 |

所有任务都支持通过 API 手动触发。

## 七、权限体系

### 7.1 角色权限

| 功能 | 普通员工 | 管理员 |
|------|---------|--------|
| 查看自己的数据 | ✅ | ✅ |
| 查看他人的数据 | ❌（除非被分享） | ✅ |
| 管理数据源 | ✅ | ✅ |
| 管理用户角色 | ❌ | ✅ |
| 查看用户列表 | ❌ | ✅ |

### 7.2 数据可见范围

员工能看到的数据 = 自己的数据 + 他人直接分享给我的 + 他人分享给我所在部门的

管理员能看到所有人的数据。

这个权限过滤在每一次数据查询、每一次搜索中都会自动执行，确保数据零越权。

## 八、部署方式

### 8.1 Docker Compose 一键部署（推荐）

```bash
# 1. 克隆代码
git clone <repo-url>
cd liuguang-data-platform

# 2. 配置环境变量
cp backend/.env.example backend/.env
# 编辑 backend/.env，填入以下必要配置：
#   - 飞书应用 ID 和密钥
#   - LLM API 密钥（DeepSeek/Qwen）
#   - JWT 签名密钥（生产环境务必修改）
#   - 数据库连接信息

# 3. 一键启动
docker-compose up -d

# 启动后包含：
#   - PostgreSQL 16 + pgvector（数据库）
#   - Backend（FastAPI 后端服务）
#   - Frontend（React 前端）
```

### 8.2 本地开发环境

```bash
# 1. 启动数据库
docker-compose up -d db

# 2. 安装 Python 依赖
cd backend
pip install -r requirements.txt

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env

# 4. 运行数据库迁移
alembic upgrade head

# 5. 启动开发服务器（支持热重载）
uvicorn app.main:app --reload
```

### 8.3 环境变量说明

| 变量 | 必填 | 说明 |
|------|------|------|
| `DATABASE_URL` | 是 | PostgreSQL 连接字符串 |
| `FEISHU_APP_ID` | 是 | 飞书应用 ID |
| `FEISHU_APP_SECRET` | 是 | 飞书应用密钥 |
| `JWT_SECRET_KEY` | 是 | JWT 签名密钥（生产环境务必修改默认值） |
| `LLM_API_KEY` | 是 | Schema 映射用的大模型 API 密钥 |
| `EMBEDDING_API_KEY` | 是 | 向量生成用的大模型 API 密钥 |
| `AGENT_LLM_API_KEY` | 是 | 智能助手用的大模型 API 密钥 |
| `FEISHU_WEBHOOK_URL` | 否 | 飞书群告警 Webhook |
| `ETL_CRON_MINUTES` | 否 | ETL 同步间隔（默认 30 分钟） |
| `UPLOAD_DIR` | 否 | 文件上传目录（默认 uploads） |
| `MAX_UPLOAD_SIZE_MB` | 否 | 最大上传大小（默认 50MB） |

## 附录：核心业务流程

### A. 用户登录流程

```
用户点击"飞书登录"
      │
      ▼
飞书授权页面 → 用户同意 → 返回授权码（code）
      │
      ▼
前端将 code 发送到 POST /api/auth/feishu/callback
      │
      ▼
后端用 code 调飞书 API，获取用户信息
      │
      ▼
在 users 表中创建或更新用户
      │
      ▼
签发 JWT（24 小时有效）→ 返回给前端
      │
      ▼
前端保存 JWT，后续请求携带 Authorization 头
```

### B. ETL 数据同步流程

```
定时任务触发（每 30 分钟）
      │
      ▼
读取所有已启用的数据源配置
      │
      ▼
对每个数据源：
  ├─ 查看上次同步时间
  ├─ 只拉取"上次之后"修改的记录（增量）
  ├─ AI 自动识别并映射字段
  ├─ 生成文本向量
  └─ 存入/更新数据库
      │
      ▼
更新同步状态（成功/失败/记录数）
```

### C. 智能问答流程

```
用户提问："最近有什么关于XX项目的会议？"
      │
      ▼
确定用户可见的数据范围（权限过滤）
      │
      ▼
同时执行向量搜索 + 关键词搜索
      │
      ▼
RRF 融合排序，取 Top N 相关内容
      │
      ▼
将相关内容 + 用户问题发给 AI 大模型
      │
      ▼
AI 基于检索到的内容生成回答
      │
      ▼
通过 SSE 流式返回给前端（逐字显示）
```
