# 流光智能数据资产平台 — 技术架构方案

> 聚焦：数据处理与AI应用
> 版本：v1.0 | 2026年3月

---

## 一、平台定位（一句话）

**流光**是一个"数据自动收集 → AI智能加工 → 知识问答与洞察"的一体化平台。
简单说：它把散落在飞书文档、会议纪要、聊天记录中的信息，自动整理成结构化的"数据资产"，并借助AI让这些数据变得"可搜索、可提问、可分析"。

---

## 二、整体架构（分层视图）

整个系统可以理解为三层"流水线"，从上到下依次是：

```
┌─────────────────────────────────────────────────────┐
│                    用户界面层                         │
│   网页端（React）— 数据看板、搜索、AI对话、报告      │
└──────────────────────┬──────────────────────────────┘
                       │ 用户操作 / 查询请求
                       ▼
┌─────────────────────────────────────────────────────┐
│                  AI 智能服务层                        │
│   RAG问答引擎 · 知识图谱 · 智能提取 · 报告生成       │
└──────────────────────┬──────────────────────────────┘
                       │ 读取 / 写入加工后的数据
                       ▼
┌─────────────────────────────────────────────────────┐
│                 数据采集与加工层                      │
│   飞书数据同步 · ETL流水线 · 向量化 · 标签系统        │

                       ▼

│                   数据存储层                          │
│   PostgreSQL数据库（支持向量检索 pgvector）           │
└─────────────────────────────────────────────────────┘
```

## 三、数据处理流程（ETL流水线）

**ETL**（Extract-Transform-Load）就是"取数据—加工数据—存数据"的过程。
平台每30分钟自动执行一次，也可以手动触发。

```
  飞书多维表格 / 文件上传 / 云文档
          │
          ▼
  ┌──────────────┐
  │ 1. 预处理器   │  ← 数据清洗：去除空白、格式统一
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 2. 提取器     │  ← 从原始数据中提取标题、内容、时间、人员等字段
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 3. 转换器     │  ← AI参与：智能识别数据类型、匹配字段映射
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 4. 增强器     │  ← AI参与：自动打标签、提取关键信息、生成摘要
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 5. 后处理器   │  ← 生成向量嵌入（用于语义搜索）、构建知识图谱
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 6. 加载器     │  ← 最终写入数据库，建立索引
  └──────────────┘
```

### 支持的数据类型

| 数据类型     | 来源               | 平台中的用途                     |
|------------|--------------------|---------------------------------|
| 文档        | 飞书文档、上传文件   | 全文检索、AI问答、知识图谱         |

## 四、AI能力矩阵

平台集成了**四类AI模型**，各司其职：

```
┌────────────────────────────────────────────────────────────┐
│                      AI 能力全景                            │
├───────────┬───────────────────┬────────────────────────────┤
│  能力类型  │     使用的模型      │         做什么              │
├───────────┼───────────────────┼────────────────────────────┤
│ 文本理解   │ GLM-4.5-air       │ ETL字段映射、关键信息提取、  │
│ 与生成     │                   │ 标签推荐、摘要生成           │
├───────────┼───────────────────┼────────────────────────────┤
│ 智能对话   │ GLM-5             │ 流光助手（AI问答）、         │
│ (Agent)   │                   │ 报告生成、日历助理           │
├───────────┼───────────────────┼────────────────────────────┤
│ 语义向量   │ BAAI/bge-m3       │ 把文本转为1024维向量，       │
│ (Embedding)│                  │ 实现"语义搜索"              │
├───────────┼───────────────────┼────────────────────────────┤
│ 视觉识别   │ Qwen3-VL-8B      │ 识别图片中的文字内容         │
├───────────┼───────────────────┼────────────────────────────┤
│ 语音转文字  │ FunASR           │ 将音频文件转写为文本         │
└───────────┴───────────────────┴────────────────────────────┘
```

> **通俗解释**：
> - **文本理解模型** = 平台的"阅读理解能力"，能看懂文档并提取重点
> - **智能对话模型** = 平台的"对话能力"，用户可以像聊天一样问它问题
> - **语义向量模型** = 平台的"联想能力"，搜索时不需要精确关键词，能理解意思相近的内容
> - **视觉/语音模型** = 平台的"眼睛和耳朵"，能看图、听录音

## 五、核心AI应用场景

### 5.1 智能问答（RAG）

**RAG**（检索增强生成）是平台最核心的AI能力。
用户提问时，系统不是凭空回答，而是先从数据库中检索相关资料，再基于资料生成回答。

```
  用户提问："上周项目进展怎么样？"
          │
          ▼
  ┌──────────────────────────┐
  │ 第1步：理解问题            │
  │ AI分析用户意图和关键词     │
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第2步：双通道检索           │
  │ ① 向量检索：找语义相似的    │  ← "意思接近"的文档
  │ ② 关键词检索：找精确匹配的  │  ← "字面一样"的文档
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第3步：融合排序(RRF)       │
  │ 把两个通道的结果合并排序    │  ← 取最相关的Top结果
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第4步：权限过滤            │
  │ 只返回用户有权限看到的数据  │  ← 数据安全保障
  └──────────┬───────────────┘
             ▼
  ┌──────────────────────────┐
  │ 第5步：AI生成回答          │
  │ 基于检索到的资料，生成回答  │  ← 附带来源引用
  │ 通过流式传输实时输出        │
  └──────────────────────────┘
```

**关键优势**：每个回答都有据可查，附带来源链接，点击即可跳转到原始文档。

### 5.2 知识图谱

平台自动从文档和会议中提取**人物、项目、组织、技术**等实体，以及它们之间的关系，构建一张"知识网络"。

```
  例如：从一份会议纪要中，AI能自动识别出——

      张三 ──负责──→ 流光项目
       │                │
      参加              属于
       │                │
       ▼                ▼
    周会          数据平台部门
```

**应用价值**：
- 可视化展示人员与项目的关联关系
- 辅助AI问答：提问"张三负责什么"时，可直接从图谱获取答案
- 领导力分析：基于图谱分析人员协作网络

### 5.3 自动待办提取

AI阅读会议纪要，自动识别其中的待办事项：

```
  会议纪要内容："下周三之前，李四完成接口文档编写"
                    │
                    ▼ AI提取
  ┌──────────────────────────────────┐
  │ 待办：完成接口文档编写              │
  │ 负责人：李四                       │
  │ 截止日期：下周三                   │
  └──────────────────────────────────┘
```

### 5.4 智能报告生成

基于平台中的数据，AI可以自动生成各类报告（周报、项目进展等），支持流式输出。

## 六、数据安全设计

```
┌──────────────────────────────────────────────────┐
│                  数据安全三道防线                   │
├──────────────────────────────────────────────────┤
│                                                  │
│  1. 身份认证：飞书SSO单点登录                      │
│     └→ 只有企业飞书账号才能访问                    │
│                                                  │
│  2. 行级权限控制：每条数据都有"归属人"              │
│     └→ 默认只能看自己的数据                        │
│     └→ 可配置部门共享、指定人共享                   │
│     └→ 管理员可查看全部                            │
│                                                  │
│  3. AI回答权限感知                                │
│     └→ AI在检索资料时，自动过滤无权限的数据         │
│     └→ 不同人问同一个问题，可能得到不同的回答       │
│                                                  │
└──────────────────────────────────────────────────┘
```

## 七、部署架构

采用 Docker 容器化部署，三个服务组成一套完整系统：

```
                    用户浏览器
                        │
                        ▼
              ┌──────────────────┐
              │   前端服务(Nginx)  │  ← 端口 80
              │   React网页应用    │
              └────────┬─────────┘
                       │ /api 请求转发
                       ▼
              ┌──────────────────┐
              │  后端服务(FastAPI) │  ← 端口 8000
              │  23个API模块      │
              │  定时ETL任务      │
              └────────┬─────────┘
                       │ 数据读写
                       ▼
              ┌──────────────────┐
              │  数据库(PostgreSQL)│  ← 端口 5432
              │  + 向量检索扩展    │
              └──────────────────┘
```

**部署特点**：
- 一键启动：`docker-compose up -d` 即可运行整套系统
- 自动迁移：启动时自动更新数据库结构
- 数据持久化：数据库数据存储在Docker卷中，不会因重启丢失

## 八、技术亮点总结

| 亮点 | 说明 |
|------|------|
| 全自动数据采集 | 每30分钟自动从飞书同步最新数据，无需人工干预 |
| AI六步流水线 | 数据经过6道AI工序加工，从原始信息变为可检索的知识资产 |
| 混合智能检索 | 语义理解 + 关键词精确匹配，双通道确保搜索质量 |
| 权限穿透到AI | 即使通过AI对话，也严格遵循数据权限，确保信息安全 |
| 知识图谱自动构建 | AI自动识别人物、项目等实体关系，无需手动维护 |
| 多模态支持 | 文档、图片、音频、视频均可处理，覆盖主要办公场景 |
| 容器化一键部署 | Docker部署简单快捷，便于环境迁移和扩展 |
